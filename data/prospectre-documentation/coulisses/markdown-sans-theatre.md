# Markdown sans théâtre

Markdown suffit largement à être ennuyeux. Il faut donc l’utiliser avec parcimonie.

## Bons usages

- un tableau pour comparer ;
- une checklist pour suivre ;
- une citation courte pour cadrer ;
- un bloc de code pour copier ;
- un lien `entity:` pour naviguer.

## Mauvais usage

Mettre tout en gras revient à ne rien mettre en gras. C’est une loi typographique, donc presque une loi morale.
