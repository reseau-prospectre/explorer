# Polir un pack existant

Le polish n’est pas une couche de vernis. C’est l’opération qui rend visible la structure réelle.

## Checklist

- Les titres visibles parlent-ils à quelqu’un qui n’a pas conçu le pack ?
- Les codes restent-ils dans les champs internes ?
- Les types fins ont-ils besoin d’être des nœuds ?
- Les sources, fragments et modèles sont-ils cachés si leur rôle est probatoire ?
- La première vue raconte-t-elle un problème, une méthode ou une carte ?

## Décision pratique

Un nœud visible doit mériter d’être sélectionné. Sinon, qu’il devienne une ligne, un tableau ou une section.
